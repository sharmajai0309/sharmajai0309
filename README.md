# 💫 About Me:
# Hey, I'm Jai Sharma 👋

> *"Engineering systems that scale silently and fail gracefully."*

I'm a Software Engineer with a deep focus on the backend systems that actually matter — the ones handling real load, real failures, and real consequences.
My work lives at the intersection of distributed systems, event-driven design, and high-throughput API engineering. I don't just write code that works; I design systems that are observable, recoverable, and built to outlast the teams that built them.
---

## ⚙️ What I Build

- **Microservices** that talk through Kafka, survive failures via DLQ & retry strategies
- **RESTful APIs** secured with JWT, optimized for concurrency and low latency
- **Distributed backends** handling 40,000+ concurrent users & 4,000 req/sec
- **Real-time systems** — spatial tracking, live telemetry, fleet coordination
- **AI-integrated platforms** — async analysis pipelines, proctored assessment engines

What I Actually Do
Event-Driven Systems & Messaging

Apache Kafka — multi-partition topic design, consumer group strategies
Retry mechanisms, Dead Letter Queue (DLQ) handling, fault isolation
Async inter-service communication at production scale

Microservices Architecture

Domain-driven service decomposition
Spring Cloud ecosystem — service discovery, config management, API gateway
Transactional atomicity across distributed pipelines (Saga pattern awareness)
Decoupled, independently deployable services with clear contracts

High-Performance APIs

RESTful API design with JWT-secured, stateless auth flows
JPA EntityGraphs, Redis caching → 35% performance uplift
Optimized for 4,000+ req/sec with thread pool tuning and query profiling

Real-Time & Spatial Systems

Live telemetry from 10,000+ concurrent delivery drivers
Geolocation API integration for radius-based discovery
Real-time fleet coordination and dynamic earnings computation

Observability & Resilience

Structured logging, error boundaries, graceful degradation
Performance profiling of JVM microservices, memory optimization
Enforcing Single Source of Truth (SSOT) across service boundaries
